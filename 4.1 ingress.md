Here’s a **complete end-to-end interview answer including AWS LoadBalancer + Ingress Controller** 👇

***

# ✅ Step 1: Dockerfile (Containerize Java App)

```dockerfile
# Build stage
FROM maven:3.9-eclipse-temurin-17 AS builder
WORKDIR /build
COPY pom.xml .
COPY src ./src
RUN mvn clean package -DskipTests

# Runtime stage
FROM eclipse-temurin:17-jdk-alpine
WORKDIR /app
COPY --from=builder /build/target/*.jar app.jar

EXPOSE 8080

ENTRYPOINT ["java", "-jar", "app.jar"]
```

***

# ✅ Step 2: Build, Tag, Push Image

```bash
docker build -t myapp .
docker tag myapp nishant123/myapp:v1
docker push nishant123/myapp:v1
```

***

# ✅ Step 3: Kubernetes Deployment

```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: java-app-deployment
  labels:
    app: java-app
spec:
  replicas: 3
  selector:
    matchLabels:
      app: java-app
  template:
    metadata:
      labels:
        app: java-app
    spec:
      containers:
      - name: java-container
        image: nishant123/myapp:v1
        ports:
        - containerPort: 8080
```

***

# ✅ Step 4: Service (ClusterIP for Ingress)

👉 IMPORTANT: When using Ingress, service should be **ClusterIP**

```yaml
apiVersion: v1
kind: Service
metadata:
  name: java-app-service
spec:
  type: ClusterIP
  selector:
    app: java-app
  ports:
    - port: 80
      targetPort: 8080
```

***

# ✅ Step 5: Install AWS Load Balancer Controller (Ingress Controller)

👉 On EKS cluster:

```bash
helm repo add eks https://aws.github.io/eks-charts
helm repo update

helm install aws-load-balancer-controller eks/aws-load-balancer-controller \
  -n kube-system \
  --set clusterName=<your-cluster-name>
```

***

# ✅ Step 6: Ingress Resource (ALB)

```yaml
apiVersion: networking.k8s.io/v1
kind: Ingress
metadata:
  name: java-app-ingress
  annotations:
    alb.ingress.kubernetes.io/scheme: internet-facing
    alb.ingress.kubernetes.io/target-type: ip
spec:
  ingressClassName: alb
  rules:
  - http:
      paths:
      - path: /
        pathType: Prefix
        backend:
          service:
            name: java-app-service
            port:
              number: 80
```

***

# ✅ Step 7: Apply All

```bash
kubectl apply -f deployment.yaml
kubectl apply -f service.yaml
kubectl apply -f ingress.yaml
```

***

# ✅ Step 8: Verify

```bash
kubectl get pods
kubectl get svc
kubectl get ingress
```

👉 You’ll get ALB DNS like:

```
http://xyz.elb.amazonaws.com
```

# ✅ Final Interview Answer (Perfect)

> "First, I containerize the Java application using a multi-stage Dockerfile and push the image to a registry. Then I deploy it to Kubernetes using a Deployment and expose it with a ClusterIP Service. I install the AWS Load Balancer Controller and create an Ingress resource with ALB annotations. This provisions an AWS Application Load Balancer, which exposes my application via a public URL."

***

# ✅ Architecture Flow

```
User → ALB (Ingress) → Service (ClusterIP) → Pods (Java App)
```
