### ✅ Answer:

**“The entrypoint is executed at runtime, not at build time.”**

***

### ✅ Explanation:

* **Build Time (Docker build phase):**
  * During this phase, the image is created using a `Dockerfile`.
  * Instructions like `RUN`, `COPY`, `ADD` are executed.
  * The goal is to **prepare the image**.

✅ Example:

```dockerfile
RUN npm install
```

This runs during build time.

***

* **Run Time (Container start phase):**
  * The **ENTRYPOINT** (and CMD) is executed when the container starts.
  * It defines the **main process** of the container.

✅ Example:

```dockerfile
ENTRYPOINT ["java", "-jar", "app.jar"]
```

This runs **when the container is started**, not during image build.

***

### ✅ Key Point for Interview:

* `ENTRYPOINT` → Executes at **runtime**
* `RUN` → Executes at **build time**

***

### ✅ Short Conclusion (Best Answer):

“ENTRYPOINT runs at container runtime, not during build. It defines the main process of the container, whereas instructions like RUN are executed during build time to prepare the image.”
